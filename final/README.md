1.你的期末專題主題簡介:

這次的期末專案我想要用進階版的to-do list，以往的to-do list非常不方便，舉例說明，如果關掉後就找不回之前的記錄。而且也沒有系統自動設定日期，要手動日期，這樣沒有防呆的機制。因此我計畫設計一個即便關掉也有記錄存在的to-do list，而且一定要配有像是google表單那種可以有系統化的選填日期的設計。 

2.使用者要怎麼使用你的網站:

使用者輸入我的github網址後，就會有相當明顯的to-do list視窗，右側為新增的區域，可以第一欄填下你預計要完成的事項。第二欄可以填寫這個事項的一些備註、註記，第三欄是我相當重視的重點，可以選擇日期。
按下Add Task之後，新的代辦事項就會跑到最左側的「To do list」欄位裡面，如果你覺得這是你「今天一定要做的」，你可以把他拉到第二欄的Must do today list裡面，讓自己知道這是必要且緊急的事件。如果你想要把「待辦事項」跟「活動」分開來看，你可以把To do list裡面的東西拉到Event today裡面，這樣可以讓自己的行程表看起來井然有序。
若您活動或是事情已經結束了，你想要把這個事件刪除，只要把方塊拉到clear data附近，就會出現一個垃圾桶，把他移到垃圾桶丟棄即可。甚至，你只要拉動方塊，注意看clear data下面已經有垃圾桶的選項出現，相當簡單又管用。
如果你要調整順序，只要將其拉動，順序就會改變。

3.你在這次專題中做了什麼，使用了什麼技術:

我這次使用了老師不斷強調的三點功能：
使用者能和網頁互動
能新增、刪除、顯示、隱藏 HTML Element 等功能
使用 jQuery 或 vue.js
就技術成面而言，我用jQuery來處理新增與刪除的部分。設計概念大致如下：每次使用者新增一個單位的事項，我則會在function裡面新增一個list。之所以用list 而不是dictionary是因為list輕鬆的變動裡面的元素，而非像dictionary一樣需要固定裡面的values。而且之後呈現在html上也可以比較方便去撰寫拖移的畫面。日期的查了很久網路的做法，後來也是使用jquery 的library去撰寫。同時把他跟新增、刪除綁在同一個class裡面，這樣code看起來比較有模組化的感覺。因為我的網站也有使用到記憶功能，也就是我的待辦事項不會因為你今天把網站關掉後，他就刪除了所以的東西。我特別去使用js來寫這樣的嵌入function，使得這樣的to-do list可以更符合使用者的需求。

4.你的網站有什麼特色、亮點，任何你覺得對評分有幫助的都可以在這邊寫上去：

 a. 主要的不同點是一般的to-do list都是一次性的，比方說你今天打開網站寫了一些東西，你關掉網站，下次再打開就不見了。相當不方便。可是我特別去改善了這樣的缺點，讓使用者即便是關掉網站、重新整理，都能保有尚未刪除之前的樣子，宛若一本真實的記事本在記錄事情。
 b. 使用了時間的選單給每個新的待辦事項。因為有了日期的選單後，可以比對日期與星期作為防呆的設計，而且畫面的一致性也可以統一，相當的有設計感。
 c. 方便的刪除設計。to-do list的變化快速，也就是說可能每幾個小時後你就完成某項任務需要刪除某個項目，傳統的to do list可能會有一個地方是刪除的按鈕來按，可是這樣的設計相當呆板。在變化快速的to do list當中，增加一些有趣的按鈕設計，可以讓畫面充滿美感與幸福感。因此我選擇了用拖拉的方式來刪除元件，可以讓使用者有一種「把事情拖到垃圾桶丟棄」的怡人心情。
 d. 顏色設計：我的主打色這次換成是灰色，灰色給人一種沈穩厚重的感覺。面對繁雜瑣碎的事項，使用了沈穩的灰色無疑讓人吃了一口定心丸，即便再怎麼多的挑戰與事情，也能夠有好的心態去面對與克服。而我的待辦事項使用白色去呈現，白色相當容易辨認，可以看清楚當下有哪件事情需要處理，也可以做到一目瞭然的效果，減緩人對複雜事情的不安定的心。
